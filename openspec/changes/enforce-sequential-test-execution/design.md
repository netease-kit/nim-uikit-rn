## Context

这次变更只涉及仓库级 agent 工作流约束，不调整页面、路由、状态或运行时配置。目标是把“测试用例逐条推进”的执行纪律写成明确规则，避免 agent 在问题尚未收敛时继续消费后续测试项。

## Decision

把该规则直接落在 `AGENTS.md` 的验证约束中，并同步更新 `agent-spec-workflow` 能力规格：

1. agent 执行测试用例时，单次只能推进一条测试用例。
2. 当前测试用例未通过时，agent 必须先修复问题并完成针对该条用例的回归验证。
3. 只有当该条用例已明确通过，或修复后的验证结果确认通过后，agent 才能进入下一条测试用例。

## Alternatives Considered

- 只在临时任务说明中口头约束：不可持续，也无法成为仓库通用规则。
- 允许批量执行后集中修复：会弱化用例与修复结果的一一对应关系，不利于定位回归来源。

## Validation Impact

这次变更属于文档和工作流规则更新，验证重点是：

- OpenSpec change 结构完整且可通过校验
- `AGENTS.md` 已明确写入串行执行与逐条验证规则
