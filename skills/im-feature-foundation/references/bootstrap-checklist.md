# IM Bootstrap Checklist

这是一份把 IM 新项目从 0 推到第一个可运行版本的启动清单。

## Phase 1: 定义范围

- 明确目标是 MVP、标准版还是完整版
- 选定首批模块
- 确认登录方式
- 确认是否需要好友 / 群组
- 确认是否有测试用例或 PRD

## Phase 2: 建立工程骨架

- 创建页面 / 路由结构
- 创建状态层目录
- 创建服务层目录
- 创建数据模型目录
- 创建 utils 和 config 目录
- 建立基础主题、错误提示和空态方案

## Phase 3: 定义契约

- 定义 User / Session / Message 数据模型
- 定义 Auth / Session / Message service 接口
- 定义登录态、会话态、消息态
- 定义错误码或错误态映射策略

## Phase 4: 先打通三条主链路

- 登录链路
- 会话列表链路
- 聊天链路

要求：
- 每条链路都至少有一个页面入口
- 每条链路都至少有一个 store 或状态容器
- 每条链路都至少有一个 service 或 SDK 抽象

## Phase 5: 补稳定性

- 本地持久化
- 网络异常处理
- 发送失败重试
- 空态 / 加载态 / 错误态
- 基础日志和调试信息

## Phase 6: 补测试

- 登录测试
- 会话测试
- 聊天测试
- 跨模块回归测试

## 默认验收门槛

最小版本通过标准：
- 可登录
- 可看会话
- 可进聊天
- 可发文本
- 可处理失败态

## 失败信号

出现以下现象，说明还不适合继续叠加高级功能：
- 登录成功但没有会话恢复策略
- 会话页和聊天页状态不一致
- 发送消息没有本地发送态
- 没有失败重试
- 没有最基础测试用例
