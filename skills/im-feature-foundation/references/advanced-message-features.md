# Advanced Message Features

用于从零规划高级消息能力。

## 覆盖范围

- 消息撤回
- 单条转发
- 合并转发
- 消息 pin
- 已读回执
- 消息操作菜单

## 模块契约

### 页面层
- 负责消息长按菜单、选择态、确认弹窗、已读详情入口、转发会话选择入口

### 状态层
- 负责消息状态变化
- 负责撤回、转发、pin、回执缓存
- 负责消息和会话列表联动

### 服务层
- 暴露 revoke / forward / pin / queryReceipts 能力

### 工具层
- 负责消息可转发性判断
- 负责合并转发负载结构
- 负责消息 key、引用关系、预览文案转换

## 功能拆解

### 1. 消息撤回

核心要求：
- 只允许满足条件的消息撤回
- 撤回后消息列表及时更新
- 会话最后一条消息预览同步更新

最小实现路径：
1. 消息菜单提供“撤回”
2. 页面触发状态层 revoke action
3. 服务层执行撤回
4. 状态层把消息改成 revoked 或展示撤回提示
5. 会话列表预览同步刷新

最少测试：
- 正常撤回
- 超时不可撤回
- 无权限撤回
- 撤回失败
- 刷新后撤回态仍正确

### 2. 单条转发

核心要求：
- 只能转发可转发消息
- 支持选择目标会话
- 转发成功后目标会话可见

最小实现路径：
1. 消息菜单提供“转发”
2. 进入会话选择页
3. 状态层执行 sendForward
4. 成功后更新最近转发目标和目标会话消息

最少测试：
- 正常转发
- 转发到多个目标
- 无可选会话
- 转发失败
- 不支持转发的消息被拦截

### 3. 合并转发

核心要求：
- 可多选消息
- 生成合并转发结构
- 目标会话展示为聊天记录卡片

最小实现路径：
1. 聊天页进入多选模式
2. 选择多条可转发消息
3. 构建 merged payload
4. 发送自定义消息
5. 详情页支持查看合并转发内容

最少测试：
- 正常合并转发
- 超过最大条数
- 包含不可转发消息
- 合并详情可查看
- 合并消息预览正确

### 4. 消息 Pin

核心要求：
- 支持 pin / unpin
- 聊天页和 pin 列表同步

最小实现路径：
1. 消息菜单提供 pin / 取消 pin
2. 状态层维护 pinned map
3. 页面可进入 pin 列表页
4. pin 状态更新时同步消息气泡展示

最少测试：
- 正常 pin
- 取消 pin
- pin 列表回显
- 刷新后 pin 状态正确

### 5. 已读回执

核心要求：
- 发送时按配置决定是否开启
- 会话或消息详情可以查看已读状态

最小实现路径：
1. 配置层提供 readReceiptEnabled 开关
2. 发送消息时附带 read receipt 配置
3. 状态层接收并缓存回执
4. 页面提供详情页查看入口

最少测试：
- 开启回执时正常上报
- 关闭回执时不展示
- 单聊回执正确
- 群回执正确
- 详情页数据正确

## 验收标准

- 高级消息操作不会破坏基础发送链路
- 消息详情页、会话页、聊天页三处表现一致
- 所有失败态都有可观察反馈
