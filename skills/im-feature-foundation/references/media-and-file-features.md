# Media And File Features

用于从零规划图片、视频、音频、文件、位置等非文本消息能力。

## 覆盖范围

- 图片消息
- 视频消息
- 音频消息
- 文件消息
- 位置消息
- 下载 / 打开 / 预览

## 模块契约

### 页面层

- 负责媒体选择入口、消息气泡渲染、预览页和详情页跳转

### 状态层

- 负责上传状态、下载状态、消息发送状态

### 服务层

- 负责上传文件、发送媒体消息、下载文件

### 工具层

- 负责权限申请
- 负责文件名解析、文件保存、本地路径管理
- 负责网络可用性检测

## 功能拆解

### 图片 / 视频

核心要求：

- 可从相册或相机选取
- 发送前完成权限检查
- 支持预览

最少测试：

- 正常发送图片
- 正常发送视频
- 权限拒绝
- 上传失败
- 预览成功

### 音频

核心要求：

- 能展示音频时长或基础信息
- 发送失败有反馈

最少测试：

- 正常发送音频
- 时长展示正确
- 发送失败重试

### 文件

核心要求：

- 可选择文件
- 可下载到本地
- 下载后可打开

最少测试：

- 正常发送文件
- 文件名和大小展示正确
- 下载成功
- 下载失败
- 打开本地文件成功 / 失败

### 位置

核心要求：

- 能发送位置消息
- 能进入位置详情

最少测试：

- 正常发送位置
- 详情页打开正确
- 缺失位置数据时有兜底

## 通用验收点

- 所有媒体消息都有发送态
- 所有下载能力都有 loading 和失败反馈
- 权限不足时不会直接崩溃
- 网络异常时给出明确提示
