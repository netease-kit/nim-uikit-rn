# 通知授权测试用例执行记录

## 执行范围

- 用例目录：`TestCases/10.0.0/通知授权`
- 已执行范围：`0824` 至 `0827`
- 执行方式：按 `AGENTS.md` 要求一次只验证一条用例；当前环境无 Detox/Maestro 等自动化 runner，采用“读取用例 -> 核对实现路径 -> 必要命令验证 -> 记录结果”的顺序。
- 环境限制：本套件依赖首次安装态、系统通知权限弹窗、系统“允许/拒绝/更多设置”按钮、系统通知设置页面以及真机通知投递结果。当前终端只能验证代码链路和 Android development build 启动状态，无法直接读取真机屏幕上的系统弹窗与通知到达结果。

## 验证命令基线

- `npm run android`：已通过。Android 真机 `PKB110` 已完成 development build 构建、安装和拉起。
- 已核对启动时通知权限请求链路：`app/_layout.tsx` 在 `preferenceStore.notificationPermissionStatus === 'undetermined'` 时自动调用 `preferenceStore.requestNotificationPermission()`。
- 已核对通知权限读写链路：`stores/PreferenceStore.ts` 通过 `utils/notifications.ts` 封装 `getPermissionsAsync()` 与 `requestPermissionsAsync()`；设置页 `app/user/notification-settings.tsx` 提供“请求授权”和“系统设置”两个入口。
