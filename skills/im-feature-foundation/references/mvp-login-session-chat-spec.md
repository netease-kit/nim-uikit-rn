# MVP Spec: Login + Session + Chat

这是一份可直接复用的 IM 最小功能集样板 spec。

适用场景：

- 新项目从零开始
- 先要一个可登录、可看会话、可发消息的最小闭环
- 后续再逐步补联系人、群组、设置等能力

## 1. 背景

需要在一个全新项目中快速建立 IM 最小闭环，确保产品具备最基本的可用性：

- 用户可以进入系统
- 用户可以看到会话
- 用户可以进入聊天并发送消息

## 2. 用户故事

- 作为用户，我希望能够登录系统，以便使用 IM 能力
- 作为用户，我希望看到我的会话列表，以便快速找到最近联系人
- 作为用户，我希望进入某个会话发送和查看消息，以便完成即时沟通

## 3. 功能范围

### 包含

- 登录
- 登录态持久化
- 会话列表
- 会话未读数展示
- 进入会话
- 历史消息拉取
- 文本消息发送
- 发送失败重试

### 不包含

- 好友申请
- 群组
- 文件消息
- 图片消息
- 消息撤回
- 转发
- pin
- 收藏

## 4. 模块契约

### 页面 / 路由层

- 登录页：手机号/验证码或账号/密码输入、错误提示、登录按钮态
- 会话页：展示会话列表、未读数、最后一条消息、进入聊天
- 聊天页：展示消息列表、输入框、发送按钮、失败重试

### 状态层

- AuthStore：登录态、登录中状态、登录恢复、登出
- SessionStore：会话列表、未读、排序、刷新
- MessageStore：消息列表、历史同步、发送中状态、失败重试

### 服务层

- AuthService：登录、登出、恢复登录态
- SessionService：获取会话列表、清理未读
- MessageService：拉取历史消息、发送消息

### 工具层

- storage：登录态持久化
- network：网络状态判断
- time：时间格式化
- message：消息预览和消息状态辅助方法

## 6. 核心实现路径

### 6.1 登录

1. 用户在登录页输入凭证
2. 页面触发 AuthStore 登录动作
3. AuthStore 调用 AuthService 完成鉴权
4. 登录成功后保存本地会话
5. 跳转会话列表页

### 6.2 会话列表

1. 进入会话页时触发 SessionStore 拉取会话列表
2. SessionStore 调用 SessionService 获取数据
3. Store 完成排序、未读聚合和展示态转换
4. 页面展示会话标题、预览、时间、未读数
5. 用户点击会话进入聊天页并清理未读

### 6.3 聊天

1. 聊天页初始化时按会话 ID 读取消息
2. 若本地未同步，则触发 MessageStore 拉历史
3. 用户输入文本并发送
4. MessageStore 先写入本地 sending 状态
5. MessageService 调用发送能力
6. 成功后更新消息状态，失败后标记 failed
7. 用户可点击失败消息重试

## 7. 实施顺序

1. 先完成工程结构、路由和基础 store 壳子
2. 完成 Auth 模块最小闭环
3. 完成 Session 模块最小闭环
4. 完成 Chat 模块最小闭环
5. 补齐错误处理和重试
6. 补齐测试用例和验收标准

## 8. 测试用例

`[TC-001] 正常登录`

- 前置条件：用户账号有效
- 操作步骤：输入凭证并点击登录
- 预期结果：登录成功并进入会话页

`[TC-002] 非法登录参数`

- 前置条件：无
- 操作步骤：输入非法手机号/验证码或错误账号信息
- 预期结果：前端拦截或接口返回错误，并展示明确提示

`[TC-003] 登录态恢复`

- 前置条件：用户已登录并存在本地会话
- 操作步骤：关闭应用后重新进入
- 预期结果：自动恢复登录态并进入主流程

`[TC-004] 拉取会话列表`

- 前置条件：账号下存在会话
- 操作步骤：进入会话页
- 预期结果：显示会话标题、时间、最后消息和未读数

`[TC-005] 空会话列表`

- 前置条件：账号下没有任何会话
- 操作步骤：进入会话页
- 预期结果：显示空态，不报错

`[TC-006] 进入会话后清未读`

- 前置条件：目标会话有未读
- 操作步骤：点击进入会话
- 预期结果：进入聊天页后该会话未读数清零

`[TC-007] 拉取历史消息`

- 前置条件：当前会话存在历史消息
- 操作步骤：进入聊天页
- 预期结果：消息列表按时间顺序展示

`[TC-008] 发送文本消息成功`

- 前置条件：网络正常
- 操作步骤：输入文本并发送
- 预期结果：消息进入 sending 态后变为成功态，列表底部更新

`[TC-009] 发送文本消息失败并重试`

- 前置条件：发送阶段模拟失败
- 操作步骤：发送消息后点击失败消息重试
- 预期结果：消息失败态可见，重试后重新进入发送流程

`[TC-010] 网络异常`

- 前置条件：断网
- 操作步骤：进入会话页或发送消息
- 预期结果：给出错误提示，不出现不可恢复卡死状态

## 9. 验收标准

- 用户可以完成登录
- 用户可以看到会话列表
- 用户可以进入会话查看消息
- 用户可以发送文本消息
- 发送失败时可以识别并重试

## 10. 后续扩展顺序

建议在 MVP 之后按以下顺序扩展：

1. 联系人
2. 群组
3. 消息撤回 / 转发
4. 文件和图片消息
5. 设置和通知
