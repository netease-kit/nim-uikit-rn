## Why

P2P chat messages generated by an AI user can be triggered from another client by mentioning the AI user. RN currently may render the trigger user as the sender for the AI reply, which makes the message attribution wrong in chat detail.

## What Changes

- Resolve AI reply message sender identity from the AI account carried by the message payload/AI metadata when available.
- Keep ordinary P2P and team sender identity resolution unchanged.
- Ensure sender avatar/name taps continue to route through the existing AI profile behavior for AI senders.

## Capabilities

### New Capabilities

### Modified Capabilities

- `chat-detail`: Chat detail identity display must attribute AI-generated reply messages to the AI user rather than the user who mentioned or triggered the AI.

## Impact

- Affected code: `app/chat/[id].tsx`, `stores/MessageStore.ts`, `src/NEUIKit/rn` message/identity adapters as needed.
- Affected behavior: Chat detail message sender name/avatar for AI reply messages.
- No new dependencies or external API changes.
