# 通讯录测试用例执行记录

## 执行范围

- 用例目录：`TestCases/10.0.0/通讯录`
- 执行方式：按 `AGENTS.md` 要求一次只验证一条用例；当前环境无 Detox/Maestro 等自动化 runner，采用“读取用例 -> 核对实现路径 -> targeted lint/类型检查 -> 记录结果”的顺序。
- 环境限制：涉及真实 IM 账号、陌生人/好友/群验证消息、多端登录、断网/切网、真机键盘和 Android 自动化标记的用例，只能记录为环境阻塞，需在真机或已配置测试账号环境复测。

## 验证命令基线

- `npx tsc --noEmit`：通过。
- `npm run lint`：失败，原因是仓库既有 Prettier/import 排序问题；其中通讯录相关 `app/contacts/blacklist-picker.tsx` 有格式问题。
- `npm run format:check`：失败，`.prettierrc` 解析报错，并有大量既有格式警告。
