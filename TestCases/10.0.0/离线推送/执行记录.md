# 离线推送测试用例执行记录

## 执行范围

- 用例目录：`TestCases/10.0.0/离线推送`
- 执行方式：按 `AGENTS.md` 要求一次只验证一条用例；当前环境无 Detox/Maestro 等自动化 runner，采用“读取用例 -> 核对实现路径 -> 必要命令验证 -> 记录结果”的顺序。
- 环境限制：本套件全部用例都依赖 iOS/Android 真机、系统通知权限、原生离线 push 通道、杀进程后的系统通知投递与通知点击回流。当前终端只能验证代码路径、运行时配置和启动前提，不能在本终端直接完成真实离线推送到达验证。

## 验证命令基线

- `npx tsc --noEmit`：通过。
- `npx eslint stores/MessageStore.ts`：通过。
- `npx eslint stores/PreferenceStore.ts stores/NIMStore.ts app/user/notification-settings.tsx utils/native-notification-settings.ts`：通过。
- `npm run start -- --non-interactive`：未形成有效启动结论。Expo CLI 明确提示 `--non-interactive is not supported, use $CI=1 instead`，随后因本机 `8081` 端口已被占用进入交互式换端口提示，当前终端无法把这次启动记为无交互启动通过。
- `npx eslint app/_layout.tsx stores/MessageStore.ts stores/PreferenceStore.ts utils/offline-push.ts utils/notifications.ts utils/native-offline-push-plugin.ts app/user/notification-settings.tsx`：失败，存在仓库当前的格式/lint 问题；其中通知相关文件有 Prettier 报错，另有 `app/_layout.tsx` hooks warning。此次测试执行未修改业务代码，相关问题按现状记录。
- `OPENSPEC_TELEMETRY=0 openspec validate align-notification-and-language-testcases --type change --no-interactive`：通过。
