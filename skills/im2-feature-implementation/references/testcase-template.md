# Test Case Template

当用户要求“测试用例”“验收点”“测试点”“回归项”时，优先使用这个模板。

## 输出原则

- 先列出直接来自现有测试用例的测试点
- 再补充边界和回归测试点
- 每条用例都能被执行和验证

## 用例模板

`[TC-001] [场景名称]`
- 来源：现有测试用例 / 新增边界用例 / 新增回归用例
- 前置条件：
- 操作步骤：
  1. 
  2. 
  3. 
- 预期结果：
  1. 
  2. 

## 建议分类

### 正常流程
- 主路径可达
- 核心动作成功
- 结果回显正确

### 边界情况
- 空数据
- 极限长度
- 重复操作
- 无权限

### 异常流程
- 接口失败
- SDK 调用失败
- 网络异常
- 数据缺失

### 回归检查
- 列表页联动
- 详情页联动
- 本地缓存 / 刷新后状态
- 邻接功能未受影响
