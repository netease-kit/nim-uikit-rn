# IM Delivery Workflow

用于指导模型如何在新项目中按功能粒度交付 IM 能力。

## 步骤

1. 明确目标是 MVP、标准版还是完整版
2. 选择一个功能模块作为最小交付单元
3. 定义该模块的页面、状态、服务、数据模型
4. 输出核心实现路径
5. 输出测试用例
6. 如果用户要求实现，再进入代码修改

## 交付原则

- 一次只推进一个功能闭环
- 先主路径，再边界和异常
- 先稳定契约，再扩功能
- 先给测试用例，再说功能完成
