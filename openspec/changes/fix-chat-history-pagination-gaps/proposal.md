# Change: 修复聊天页历史消息分页缺失

## Why

当前 RN 聊天页拉取历史消息时，与 Android、iOS 原生端的分页参数不一致。RN 首屏只加载较少消息，向上翻页时也没有携带与原生一致的时间边界，导致部分历史记录在分页边界附近出现断续缺失。

## What Changes

- 将 RN 聊天页首屏历史消息页大小对齐原生端的默认分页数量。
- 将 RN 向上加载历史消息的分页参数对齐原生/Web 基线，分页时同时携带锚点消息和时间边界。
- 保持现有消息顺序、上滑预取时机和聊天页交互不变，只修复历史消息完整性。

## Impact

- 影响 `stores/MessageStore.ts` 的历史消息拉取逻辑。
- 影响聊天页首次进入和继续上滑查看历史时的消息完整性。
- 不改动发送、已读、置顶消息和页面样式行为。
