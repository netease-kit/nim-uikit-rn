## Why

云端会话登录后，底部消息 Tab 的红点当前依赖已展示会话列表的未读汇总。若第一页没有未读、后续分页存在未读，用户在首屏登录完成后看不到红点，只有上滑加载后续分页后才会出现，和全量未读状态不一致。

## What Changes

- 将底部消息 Tab 红点改为基于当前会话源维护的全量未读总数，而不是当前已展示分页的未读和。
- 将应用前后台未读数同步统一改为相同的全量未读来源，避免云端分页场景下 SDK 侧未读同步滞后。
- 保持会话列表分页、未读清零和免打扰展示逻辑不变。

## Impact

- 受影响代码：`app/(tabs)/_layout.tsx`、`app/_layout.tsx`
- 受影响行为：云端会话模式下消息 Tab 红点与应用未读同步
- 无新增依赖，无接口协议变更
